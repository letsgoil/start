02. Git을 설치하고 로컬저장소에서 커밋 관리하기

  1. git 이란 무엇인가 하면 버전을 관리하는 프로그램이다.
  2. git bash 라는 터미널을 이용해서 디렉토리로 이용하여 모든 명령을 내린다.
  3. 커밋이란 무엇인가 하면 하나하나의 버전을 나타내는 history 단계를 나타낸다.
  4. $ git init :현재 디렉토리를 git으로 버전 관리한다는 의미
  5. $ git config --global user.email "letsgoil12@gmail.com"
  6. $ git config --global user.name "letsgoil"
  7. $ git add README.txt : 커밋에 포함시킬 파일이나 디렉토리를 지정해 준다.
  8. $ git commit -m "현재버전 커밋 설명 이걸보고 이 커밋으로 찾아온다." : 하나의 커밋을 history로 저정하는 명령임
  9. $ git log : 현재까지 생성된 커밋을 보여준다. 이걸보고 찾아갈 커밋을 인지한다.
  10. $ git checkout 5813bb5 : git log 에서 확인한 커밋의 번호를 가지고 그단계로 이동한다.
  11. $ git checkout - : 가장최근의 history 단계로 복귀한다.

03. GitHub 원격저장소에 커밋 올리기

  1. 원격저장소 : GitHub에 생성한 Repository
  2. 로컬저장소 : 내컴퓨터에서 생성한 디렉토리로 $ git init 를 실행한 디렉토리
  3. 원격저장소에 로컬저장소의 커밋을 올리는 것이다. history상의 어떤 커밋을 올리는 것이다.
  4. GitHub에 생성된 원격저장소를 레포지토리 Repository 라고 부른다.
  5. New repository 생성 [Repository name] : 생성될 디렉토리명 이라고 이해 [Description] : 디렉토리 설명 [Create repository] : 생성
  6. 생성한 원격저장소 주소 https://github.com/letsgoil/start.git 끝부분의 start 가 생성한 Repository 이다.
  7. 원격저장소에 커밋 올리기려면 GitHub에서 만든 Repository 주소를 내컴퓨터의 로컬저장소(git init 한디렉토리)에 알려주고 커밋을 원격저장소에 올린다.
  8. $ git remote add origin https://github.com/letsgoil/start.git : 로컬저장소에 원격저장소 주소를 알려주는것
  9. $ git push origin master : 원격에 마스터 커밋을 올리는 것

04. GitHub 원격저장소의 커밋을 로컬저장소에 내려 받기

  1. 내려받을 로컬저장소를 생성하고 git bash 로 이동해서 명령을 내린다.
  2. 원격저장소의 주소는 GitHub 에서 [Clone or download] 버튼을 눌러서 주소를 복사한다.
  3. git clone https://github.com/letsgoil/start.git . : 마지막에 한칸띄우고 . 점을 찍어야 현재 디렉토리에 바로 받는다 없으면 start 디렉토리를 생성하고 넣는다.
  4. 다운받은 커밋을 수정해서 다시 새로운 커밋을 생성한다
  5. $ git add README.txt
  6. $ git commit -m "개발자목록추가"
  7. $ git push orgin master : 로컬저장소의 커밋을 원격저장소에 올리는것을 push 라한다.
  8. 내 로컬저장소로 원격저장소에 새로운 커밋이 있으면 받아라는 명령을 해보자
  9. $ git pull origin master : git clone으로 통으로 받아와서 push 와 pull 로 넣고 당겨오고 한다.
